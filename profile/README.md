## What is Simpson Systems?

**Simpson Systems** is a collection of projects focused on understanding how software actually works — not just frameworks, but the layers underneath them.

The organization exists as both:

• a software lab
• a reference implementation of disciplined engineering practices

Every repository is intended to either:

1. build a real system
2. improve how systems are built

---

## Areas of Work

### Systems Programming

Exploring the lower layers of computing.

* operating system and kernel experiments
* memory management
* virtualization environments
* runtime behavior

### Developer Tooling

Making good engineering habits easier to follow.

* workflow automation
* repository bootstrap tools
* commit and pull request structure
* project traceability

### Cloud & Infrastructure

Reproducible environments and predictable deployments.

* CI/CD pipelines
* local cloud environments
* service orchestration
* infrastructure experimentation

---

## Engineering Principles

We treat process as part of the product.

* Small changes over large changes
* Every change traceable to a reason
* Automation over manual steps
* Reproducible environments
* Documentation lives with the code
* Tools should teach, not just enforce

---

## Why This Exists

Many developers learn languages and frameworks, but not how professional software engineering operates in practice.

This organization is an attempt to make those practices concrete:
a working example of how ideas like code review, CI, reproducibility, and structured development actually look in a real project.

---

## How to Explore

Each repository documents:

* how to build it
* how to run it
* what problem it solves

Start with any repository README and its `docs/` directory.
